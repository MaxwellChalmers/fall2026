# Lab Index Context

## Core SYS labs

These are the main labs for the semester-long SYS course.

|   # | Lab                                   | Core purpose                                                        |
| --: | ------------------------------------- | ------------------------------------------------------------------- |
|   1 | Classification Is Not Neutral         | Categories and labels are social and technical decisions.           |
|   2 | Training Data Shapes Learning         | Models learn from examples, labels, and omissions.                  |
|   3 | Features Are Value Choices            | Variables and proxies encode values and institutional choices.      |
|   4 | Thresholds and the Cost of Error      | Scores become decisions through thresholds that distribute harm.    |
|   5 | Prediction and Historical Data        | Predictive systems can reproduce historical inequality.             |
|   6 | Neural Networks and Opacity           | Layered systems raise explainability and contestability questions.  |
|   7 | Sensors Translate the World into Data | Data is produced through measurement and sensing.                   |
|   8 | Anomaly Detection and Normal          | Systems define baselines and flag difference as risk.               |
|   9 | Edge AI, Privacy, and Accountability  | On-device AI changes data flows, oversight, and responsibility.     |
|  10 | BRAID Anticipatory Governance Lab     | Students synthesize the course through an emerging technology case. |

## BRAID / hardware extension labs

These support the BRAID resource hub and optional deeper technical modules.

|   # | Lab                                          | Core purpose                                                                                    |
| --: | -------------------------------------------- | ----------------------------------------------------------------------------------------------- |
|  11 | Memory as Physical State                     | Memory, storage, deletion, and learning are physical and governable.                            |
|  12 | Von Neumann vs. Neuromorphic Architectures   | Architecture shapes energy, speed, auditability, and control.                                   |
|  13 | Spikes, Timing, and Event-Driven Computation | Spiking systems use timing and events to process information.                                   |
|  14 | Hardware Governance Lab                      | Hardware-level AI requires patching, logging, auditing, repair, and lifecycle governance.       |
|  15 | Energy and Computation                       | Energy efficiency claims must be evaluated across lifecycle, scale, and environmental justice.  |
|  16 | Learning, Memory, and Biological Metaphors   | Biological metaphors can clarify mechanisms but mislead about intelligence, agency, and ethics. |
|  17 | Electricity as Information                   | Physical states become bits, signals, and data.                                                 |
|  18 | How Circuits Do Math                         | Logic gates and circuits implement arithmetic and computation.                                  |

## Pedagogical sequence

The core sequence is:

classification → data → features → thresholds → history → opacity → measurement → anomaly detection → edge AI → anticipatory governance

The BRAID extension sequence is:

physical signals → circuits → memory → architecture → spikes/timing → hardware governance → energy/environment → biological metaphor critique
